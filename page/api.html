API jquery.windfury.js

Windfury File Pattern

The main structure of windfury must exist in a structure, where we have a root tag section with a class windfury.
Inside the section tag must exist a script tag for this file be declared as windfury file. Something like this: 

<section class="windfury">
    <script type="text/javascript">
        [Component Code]
    </script>
</section>

Then you can add more sections creating snippet templates, like this: 

<section class="windfury">
    <section class="template">
        <div>
            [Html Template]
        </div>
    </section>

    <script type="text/javascript">
      [Component Code]
    </script>
</section>

The tag script will be executed and you will have a object named as windfury, which has the group of following functions:

  • windfury.def(anyvalue):

    The value defined here will be receive as return of this components, can be any value.

  • windfury.read(clazz):

    This function will look over the snippets templates inside the windfury file.
    The return will be a text of the inner Html of section that you were looking for.

  • windfury.text(clazz):

    Is the same as windfury.read, but this one will return the text encapsulated inside a function.

    This function is for compatibility purpose.

  • windfury.doT(clazz):

    Is the same as windfury.read, but it will return a snippet compiled as a doT function.
    This is added to help create templates from json.

  • windfury.req(array, successCallback):

    This is a helper function created to isolated the scope of the script tag, it's first parameters is an array where you can pass all urls of windfury templates you want to find.
    The second parameter is a function the first parameter will be the jQuery and the second will be the windfury, after the will come all the objects defined inside the templates asked for in the array.
    When using this function you can call a function $.wf.autoloads(array);, the array parameters is filled with urls of windfury files, the idea is when you call the req function after jQuery parameter will come the objects of autoload and only after that will come the objects where you require.

$.windfury(windfuryPattern, successCallback)

  • windfuryPattern: this is the text pattern where must have a section tag root with a class windfury, inside the tag must have a script section

  • successCallback: this a function where you will receive the value where you defined in the section script.

$.getWindfury(url, data, success, error)

  • url: is the url of a windfury templates
  • data [optional field]: is values if you and to send anything to the ajax needed to ask for the windfury template
  • success: function where will be called after the template is parsed and executed, this function you will receive the parameters you define in the def function.
  • error [optional field]: if in any reason the ajax for getting the template isn't workout this function will be called.

$.wf(urls, success, error)

  • urls: is a array of urls with windfury templates
  • success: function where will be called after all templates are parsed and executed, this function you will receive all the parameters you define in the def function in the array order.
  • error [optional field]: if in any reason the any of the ajax for getting a template isn't workout this function will be called.
